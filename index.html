Digital Pressure Monitoring Cluster (V1 - MVP)

Designed and developed a real-time embedded system for railway applications to monitor multiple pressure sensors and visualize data with high responsiveness and reliability.

Key Highlights

  • STM32-based embedded system with optimized single-loop architecture (no RTOS)
  • Deterministic loop cycle of 135 ms for real-time performance
  • Supports 6 CAN-based pressure sensors
  • Pressure monitoring up to 11 bar
  • Periodic data logging every 3 seconds to SD card

Interfaces Used

  • CAN → Pressure sensors
  • UART → Industrial HMI display
  • SPI → SD card storage
  • I2C → Real-time clock (RTC)
  • USB → Firmware update & data transfer

Features

  • Real-time pressure gauge visualization
  • Warning, error, and system status indication
  • Robust USB firmware update with fallback mechanism
  • USB-based data extraction from SD card
  • Multi-function control using only 4 hardware buttons

Your Contribution

  • Designed complete firmware architecture and system flow
  • Implemented CAN communication for multi-node sensor network
  • Developed advanced multi-function button handling logic
  • Integrated SD card logging and USB data transfer
  • Optimized execution timing to achieve deterministic 135 ms loop

Digital Pressure Monitoring Cluster – Advanced (V2)

Led the development of a next-generation embedded system evolving from a bare-metal architecture to a scalable RTOS-based platform with advanced GUI, external memory, and industrial-grade reliability.

Key Improvements Over V1

  • FreeRTOS-based multitasking architecture
  • External QSPI Flash for application execution
  • External RAM for GUI and buffering
  • eMMC storage via SDMMC interface
  • LVDS-based high-resolution display with TouchGFX
  • Support for multiple sensor vendors with auto-detection

System Architecture

  • Internal Flash → Bootloader with fallback support
  • External Flash (QSPI) → Main firmware execution
  • External RAM → GUI rendering & runtime buffers
  • eMMC → Data logging
  • CAN / CAN-FD → Sensor communication
  • LVDS → Display interface

Engineering Challenges

  • Executing firmware from external flash (QSPI)
  • Designing robust bootloader with fallback mechanism
  • Balancing RTOS tasks for GUI and communication
  • Integrating TouchGFX for high-performance UI
  • Handling multiple sensor protocols dynamically
  • Implementing reliable eMMC data storage

Your Role

  • Technical Lead for firmware architecture
  • Defined system design and memory layout
  • Led team development and task distribution
  • Handled complex modules (RTOS, memory, GUI)
  • Mentored team and resolved critical issues

Development & Hardware

V1 vs V2 Comparison

  • Architecture: Bare-metal → FreeRTOS
  • Storage: SD Card → eMMC
  • UI: Basic → Advanced GUI (TouchGFX)
  • Memory: Internal → External Flash + RAM
  • Scalability: Limited → High

Impact: Delivered a scalable, production-grade embedded system with improved UI, flexibility, and reliability.
